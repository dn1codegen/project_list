# Инструкции по установке и использованию

Rails 8 приложение для управления проектами с клиентской загрузкой файлов.

## Требования

- Ruby 3.4.8+
- SQLite3
- RubyGems

## Установка

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd list
```

### 2. Установка зависимостей

```bash
bundle install
```

### 3. Настройка базы данных

```bash
bin/rails db:setup
```

Или по шагам:
```bash
bin/rails db:create
bin/rails db:migrate
bin/rails db:seed  # если есть сиды
```

### 4. Создание администратора

```bash
rails runner "Admin.create!(email: 'admin@example.com', password: 'admin123', password_confirmation: 'admin123')"
```

⚠️ **Важно:** Обязательно создайте администратора перед первым запуском!

### 5. Запуск сервера

```bash
bin/rails server
```

Сервер будет доступен по адресу: `http://localhost:3000`

## Веб-интерфейс

### Администратор

**Вход в админку:**
1. Перейдите на: `http://localhost:3000/admin/login`
2. Введите email: `admin@example.com`
3. Введите пароль: `admin123`
4. Нажмите "Войти"

**Функции админки:**
- 📊 Дашборд со списком всех ключей
- 🔑 Создание ключей для веб-доступа
- 🗝️ Создание ключей для проектов (API)
- 📈 Просмотр статистики по каждому пользователю
- ⚡ Активация/деактивация ключей
- 🗑️ Удаление ключей
- ⚙️ Настройки профиля (смена email и пароля)

### Пользователи

Пользователи получают доступ через уникальные токены, которые создает администратор.

**Доступ к проектам:**
```
http://localhost:3000/projects?token=ВАШ_ТОКЕН
```

**Просмотр конкретного проекта:**
```
http://localhost:3000/projects/1?token=ВАШ_ТОКЕН
```

## Создание и управление ключами

### Ключ для веб-доступа

Используется для доступа пользователей к веб-интерфейсу:

1. Войдите в админку
2. В разделе "Ключи веб-доступа" нажмите "Создать"
3. Введите название (например: имя пользователя)
4. Скопируйте URL из карточки ключа
5. Отправьте ссылку пользователю

### Ключ для проектов (API)

Используется клиентским скриптом для загрузки проектов:

1. Войдите в админку
2. В разделе "Ключи для проектов" нажмите "Создать"
3. Введите название
4. Скопируйте ключ
5. Используйте в `client/project_config.yaml`

Подробнее в документации: [API_KEYS.md](API_KEYS.md)

## Клиентский скрипт

### Установка клиента

Скрипт находится в папке `client/`.

**Требования:**
- Ruby 2.7+
- YAML (входит в стандартную библиотеку)
- MIME::Types

Установка MIME::Types:
```bash
gem install mime-types
```

### Настройка клиента

1. Перейдите в папку клиента:
   ```bash
   cd client
   ```

2. Создайте конфигурационный файл:
   ```bash
   cp project_config.example.yaml project_config.yaml
   ```

3. Отредактируйте `project_config.yaml`:
   ```yaml
   server_url: "http://localhost:3000"
   api_key: "ВАШ_КЛЮЧ_ДЛЯ_ПРОЕКТОВ"

   project:
     sku: "TEST-001"           # Уникальный артикул
     name: "Название проекта"
     customer: "Заказчик"
     address: "Адрес"
     description: |
       Описание проекта
       - Пункт 1
       - Пункт 2

   directories:
     measurements: "замеры"      # Папка с фотографиями замеров
     examples: "примеры"         # Папка с примерами
     project_pdf: "проект"       # Папка с PDF файлом
   project_pdf_description: "Описание изменений"
   ```

4. Разместите файлы в соответствующих папках:
   ```
   client/
   ├── замеры/
   │   ├── фото1.jpg
   │   └── фото2.jpg
   ├── примеры/
   │   ├── пример1.jpg
   │   └── пример2.jpg
   └── проект/
       └── проект.pdf
   ```

### Отправка проекта

```bash
cd client
ruby send.rb
```

Или с указанием конфига:
```bash
ruby send.rb my_project.yaml
```

Подробнее в документации: [client/README.md](client/README.md)

## API

### Создание проекта

**POST** `/api/projects`

**Headers:**
```
Authorization: Bearer ВАШ_API_КЛЮЧ
Content-Type: multipart/form-data
```

**Body:**
- `project[sku]` - уникальный артикул
- `project[name]` - название проекта
- `project[customer]` - заказчик
- `project[address]` - адрес
- `project[description]` - описание
- `measurements[][file]` - файлы замеров
- `measurements[][description]` - описания замеров
- `examples[][file]` - файлы примеров
- `examples[][description]` - описания примеров
- `project_pdf[file]` - PDF проекта
- `project_pdf[description]` - описание PDF

## Статистика и логирование

Система автоматически отслеживает:

- ✅ Время входа пользователя
- ✅ Просмотры каждого проекта
- ✅ Длительность просмотра
- ✅ IP адрес и User-Agent

**Просмотр статистики:**
1. Войдите в админку
2. В карточке ключа веб-доступа нажмите "Статистика"
3. Увидите таблицу с информацией по каждому проекту

## Безопасность

### Токены и ключи

- 🔐 Ключи генерируются автоматически
- 🔑 Каждый ключ уникален
- ⚡ Ключи можно активировать/деактивировать
- 🗑️ Ключи можно удалить

### Роли доступа

**Администратор:**
- Полный доступ ко всем функциям
- Управление ключами
- Просмотр статистики
- Настройки профиля

**Пользователь:**
- Доступ только к своим проектам
- Просмотр информации о проектах
- Скачивание PDF файлов

### Важные правила безопасности

⚠️ **Никогда не публиковать**:
- Ключи в открытых репозиториях
- Ключи в чатах
- Ключи в публичной документации

✅ **Рекомендуется:**
- Регулярно менять ключи
- Использовать уникальные ключи для разных пользователей
- Деактивировать неиспользуемые ключи
- Хранить ключи в безопасном месте

## Устранение проблем

### Не работает вход в админку

**Проблема:** Невозможно войти
**Решение:**
1. Убедитесь, что администратор создан:
   ```bash
   rails runner "puts Admin.count"
   ```
2. Если администраторов нет - создайте:
   ```bash
   rails runner "Admin.create!(email: 'admin@example.com', password: 'admin123', password_confirmation: 'admin123')"
   ```
3. Проверьте правильность email и пароля

### Не работает отправка проекта

**Проблема:** Скрипт send.rb выдает ошибку
**Решение:**
1. Проверьте, что сервер запущен: `bin/rails server`
2. Убедитесь, что используете правильный тип ключа (для проектов)
3. Проверьте, что ключ активен в админке
4. Проверьте конфигурацию `client/project_config.yaml`
5. Убедитесь, что файлы существуют в указанных папках

### Не загружаются файлы

**Проблема:** Файлы не загружаются или не отображаются
**Решение:**
1. Проверьте права доступа к папкам
2. Убедитесь, что файлы существуют
3. Проверьте размер файлов (лимит зависит от настроек Rails)
4. Проверьте логи сервера: `bin/rails server`

### Нет статистики

**Проблема:** Статистика не отображается
**Решение:**
1. Убедитесь, что пользователь заходит по ссылке с токеном
2. Проверьте, что ключ для веб-доступа активен
3. Проверьте, что пользователь действительно открывал проекты
