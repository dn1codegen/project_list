# Информация об API ключах

## Два типа API ключей

Система поддерживает два типа API ключей для разных целей:

### 1. Ключ для отправки проектов (API)

**Назначение:** Отправка проектов через клиентский скрипт (send.rb)

**Где использовать:**
- В файле `client/project_config.yaml`
- В параметре `api_key` при вызове скрипта

**Пример:**
```bash
cd client
ruby send.rb project_config.yaml
```

### 2. Ключ для веб-доступа (по ссылке)

**Назначение:** Просмотр проектов через веб-интерфейс по ссылке

**Где использовать:**
- В URL как параметр `?token=...`
- При открытии проектов в браузере

**Пример:**
```
http://localhost:3000/projects/1?token=ВАШ_КЛЮЧ
```

## Создание ключей через админку

### Вход в админку

1. Откройте страницу входа: `http://localhost:3000/admin/login`
2. Введите email и пароль администратора
3. Нажмите "Войти"

### Создание ключа для веб-доступа

1. На дашборде найдите раздел **"Ключи веб-доступа"**
2. Введите название ключа (например: "Василий")
3. Нажмите **"Создать"**
4. Скопируйте сгенерированный ключ и URL
5. Отправьте ссылку пользователю

### Создание ключа для отправки проектов

1. На дашборде найдите раздел **"Ключи для проектов"**
2. Введите название ключа (например: "API для проекта 1")
3. Нажмите **"Создать"**
4. Скопируйте сгенерированный ключ
5. Используйте его в `client/project_config.yaml`

## Управление ключами

### Карточки ключей

В админке ключи отображаются в виде карточек с следующей информацией:

**Для ключей веб-доступа:**
- 📛 Название ключа (например: "Василий")
- 🔗 URL для доступа (с токеном)
- ⚡ Статус (активный/неактивный)
- 📊 Кнопка "Статистика" - просмотры проектов
- 🗑️ Кнопка "Удалить" - удаление ключа

**Для ключей проектов:**
- 📛 Название ключа
- 🔑 API ключ
- ⚡ Статус (активный/неактивный)
- 🗑️ Кнопка "Удалить" - удаление ключа

### Активация/деактивация ключей

Переключатель в карточке ключа позволяет:
- **Включить** - ключ активен и работает
- **Выключить** - ключ отключен, доступ запрещен

### Просмотр статистики

Для ключей веб-доступа доступна статистика:

1. Нажмите кнопку **"Статистика"** в карточке ключа
2. Увидите таблицу с информацией:
   - Название проекта
   - Количество просмотров
   - Последнее посещение

### Удаление ключа

1. Нажмите кнопку **"Удалить"** в карточке
2. Подтвердите удаление

⚠️ **Важно:** После удаления ключа доступ будет невозможен!

## Типы ключей

| Тип | Использование | Где применять |
|-----|---------------|----------------|
| 🔑 Отправка | Скрипт send.rb | `client/project_config.yaml` |
| 🌐 Веб-доступ | Браузер | URL `?token=...` |

## Примеры использования

### Отправка проекта через API

1. Создайте ключ для проектов в админке
2. Скопируйте ключ
3. Отредактируйте `client/project_config.yaml`:
   ```yaml
   server_url: "http://localhost:3000"
   api_key: "ВАШ_КЛЮЧ"
   ```
4. Запустите скрипт:
   ```bash
   cd client
   ruby send.rb project_config.yaml
   ```

### Просмотр проекта по ссылке

1. Создайте ключ для веб-доступа в админке
2. Скопируйте URL из карточки (он уже содержит токен)
3. Отправьте ссылку пользователю

Или вручную добавьте токен к URL:
```
http://localhost:3000/projects/1?token=ВАШ_КЛЮЧ
```

### Просмотр списка проектов

```
http://localhost:3000/projects?token=ВАШ_КЛЮЧ
```

## Управление через консоль (для разработчиков)

### Создание ключей

```bash
# Ключ для веб-доступа
rails runner "WebAccessKey.create!(name: 'Тестовый пользователь', key: SecureRandom.hex(32), active: true)"

# Ключ для проектов
rails runner "ProjectKey.create!(name: 'API ключ', key: SecureRandom.hex(32), active: true)"
```

### Просмотр всех ключей

```bash
# Веб-доступа
rails runner "WebAccessKey.all.each { |k| puts \"#{k.name}: #{k.active ? '✅' : '❌'}\" }"

# Для проектов
rails runner "ProjectKey.all.each { |k| puts \"#{k.name}: #{k.active ? '✅' : '❌'}\" }"
```

### Активация/деактивация

```bash
# Активировать все ключи веб-доступа
rails runner "WebAccessKey.update_all(active: true)"

# Деактивировать все ключи
rails runner "WebAccessKey.update_all(active: false); ProjectKey.update_all(active: false)"
```

## Безопасность

### ⚠️ Важные правила безопасности

- 🔐 **НЕ ПУБЛИКУЙТЕ** ключи в открытых репозиториях
- 📋 **Храните** ключи в безопасном месте
- 🔄 **Регулярно обновляйте** ключи
- 🚫 **НЕ ИСПОЛЬЗУЙТЕ** один ключ для разных проектов
- 🔑 **Используйте разные ключи** для разных пользователей
- ⚡ **Деактивируйте** ключи при подозрении на утечку

### Разделение доступа

Ключи для отправки и веб-доступа разделены:

- **Ключ для отправки** - только для API (скрипт send.rb)
- **Ключ для веб-доступа** - только для браузера (пользователи)

⚠️ **Важно:** Не используйте ключ для отправки в качестве токена веб-доступа и наоборот!

## Токены доступа

В системе используются два способа аутентификации:

### 1. Токены веб-доступа

Автоматически генерируются при создании ключа веб-доступа.
Используются в URL: `?token=...`

### 2. API ключи

Используются для аутентификации при отправке проектов через API.
Передаются в заголовке `Authorization: Bearer ...` или в конфигурационном файле.

## Устранение проблем

### Ключ не работает

1. Проверьте, что ключ активирован в админке
2. Убедитесь, что используете правильный тип ключа
3. Проверьте, что ключ полностью скопирован

### Нет доступа к проектам

1. Убедитесь, что используете правильный URL с токеном
2. Проверьте, что ключ для веб-доступа активен
3. Проверьте консоль браузера на ошибки

### Ошибка API

1. Убедитесь, что используете ключ для проектов
2. Проверьте конфигурацию `client/project_config.yaml`
3. Убедитесь, что сервер запущен: `bin/rails server`
