# Клиент для отправки проектов

## Структура директорий

```
client/
├── send.rb                       # Скрипт отправки проекта
├── project_config.yaml           # Основной файл конфигурации
├── project_config.example.yaml    # Пример конфигурации
├── README.md                     # Документация
├── замеры/                        # Папка с файлами замеров (изображения)
├── примеры/                       # Папка с файлами примеров (изображения)
└── проект/                        # Папка с PDF файлами проекта
```

## Быстрый старт

1. **Отредактируйте конфигурацию**
   ```bash
   # Скопируйте пример конфигурации (если нужно)
   cp project_config.example.yaml project_config.yaml

   # Отредактируйте project_config.yaml:
   # - Укажите server_url
   # - Укажите api_key
   # - Заполните данные проекта
   # - Укажите пути к файлам
   ```

2. **Разместите файлы**
   ```
   client/
   ├── замеры/
   │   ├── фото1.jpg
   │   └── фото2.jpg
   ├── примеры/
   │   ├── пример1.jpg
   │   └── пример2.jpg
   └── проект/
       └── проект.pdf
   ```

3. **Отправьте проект**
   ```bash
   cd client
   ruby send.rb
   ```

Или с указанием другого файла конфигурации:
```bash
ruby send.rb my_project.yaml
```

## Формат конфигурации

### Основные настройки

```yaml
# URL сервера
server_url: "http://localhost:3000"

# API ключ (получите через веб-интерфейс на странице /api_keys)
api_key: "ВАШ_API_КЛЮЧ"

# Данные проекта
project:
  sku: "TEST-001"                # Уникальный артикул для обновления
  name: "Название проекта"
  customer: "Заказчик"
  address: "Адрес"
  description: |
    Многострочное описание проекта
    - Пункт 1
    - Пункт 2
```

### Директории с файлами

```yaml
directories:
  # Замеры (изображения) - скрипт автоматически загрузит все файлы из папки
  measurements: "замеры"

  # Примеры (изображения) - скрипт автоматически загрузит все файлы из папки
  examples: "примеры"

  # Проект в формате PDF - скрипт автоматически загрузит первый PDF файл из папки
  project_pdf: "проект"
  project_pdf_description: "Последние изменения проекта"
```

## Получение API ключа

1. Откройте в браузере: `http://localhost:3000/api_keys`
2. Нажмите "Создать новый API ключ"
3. Введите название и нажмите "Создать"
4. Скопируйте сгенерированный ключ в конфигурацию

## Обновление проекта

При повторной отправке с тем же `sku`, проект будет обновлен:
- Текстовые данные заменятся на новые
- Файлы перезагрузятся
- Счетчик обновлений увеличится

## Важные примечания

- **Автоматическая загрузка файлов**: Скрипт автоматически загружает все файлы из указанных папок
- **Автоматические описания**: Для всех файлов описание берется автоматически из имени файла (без расширения)
- **Описание PDF**: Для проекта (PDF) можно указать описание отдельно (project_pdf_description)
- **Уникальный SKU**: Используйте уникальный артикул для каждого проекта
- **Размеры файлов**: Учитывайте ограничения на размер файлов при загрузке
- **Типы файлов**:
  - `замеры/` - все файлы будут загружены как замеры
  - `примеры/` - все файлы будут загружены как примеры
  - `проект/` - первый PDF файл будет загружен как проект
- **Исключение**: Файлы с "Zone.Identifier" в имени игнорируются

## Возможности интерфейса

✅ **Карточки изображений** - замеры и примеры отображаются в красивых карточках
✅ **Модальное окно** - клик по карточке открывает увеличенное изображение
✅ **Названия без расширений** - чистые названия файлов без .jpg, .png
✅ **PDF в информации** - PDF проекта отображается в блоке "Информация о проекте"
✅ **Счетчик обновлений** - автоматическое отслеживание количества обновлений проекта

## Примеры использования

### Отправка нового проекта
```bash
ruby send.rb project_config.yaml
```

### Обновление существующего проекта
```bash
# Просто отправьте с тем же sku
ruby send.rb project_config.yaml
```

### Отправка с другим конфигом
```bash
ruby send.rb client/project_config.yaml
```

## Устранение проблем

### Файл конфигурации не найден
```
Ошибка: Файл конфигурации не найден: project_config.yaml
```
**Решение:** Убедитесь, что находитесь в директории `client/` или укажите полный путь

### API ключ недействителен
```
✗ Ошибка:
{"error": "Invalid API key"}
```
**Решение:** Получите новый API ключ на странице `/api_keys`

### Файл не найден
```
  ✗ замеры/фото.jpg - ФАЙЛ НЕ НАЙДЕН!
```
**Решение:** Проверьте, что файл существует в указанной папке относительно директории `client/`

## Требования

- Ruby 2.7+
- YAML (входит в стандартную библиотеку)
- MIME::Types (установите через: `gem install mime-types`)
