# Клиент для отправки проектов

Скрипт для отправки проектов на сервер через API.

## Особенности

- ✅ Автоматическая загрузка всех файлов из папок
- ✅ Автоматические описания из имен файлов
- ✅ Поддержка обновления проектов по SKU
- ✅ Простая конфигурация через YAML
- ✅ Поддержка изображений и PDF

## Структура директорий

```
client/
├── send.rb                       # Скрипт отправки проекта
├── project_config.yaml           # Основной файл конфигурации
├── project_config.example.yaml   # Пример конфигурации
├── README.md                     # Документация
├── замеры/                        # Папка с файлами замеров (изображения)
├── примеры/                       # Папка с файлами примеров (изображения)
└── проект/                        # Папка с PDF файлами проекта
```

## Быстрый старт

### 1. Получите API ключ

API ключ создается в админке:

1. Войдите в админку: `http://localhost:3000/admin/login`
2. В разделе "Ключи для проектов" нажмите "Создать"
3. Введите название и нажмите "Создать"
4. Скопируйте сгенерированный ключ

### 2. Настройте конфигурацию

```bash
cd client
cp project_config.example.yaml project_config.yaml
```

Отредактируйте `project_config.yaml`:
```yaml
server_url: "http://localhost:3000"
api_key: "ВАШ_API_КЛЮЧ"

project:
  sku: "TEST-001"                # Уникальный артикул для обновления
  name: "Название проекта"
  customer: "Заказчик"
  address: "Адрес"
  description: |
    Многострочное описание проекта
    - Пункт 1
    - Пункт 2

directories:
  measurements: "замеры"         # Папка с фотографиями замеров
  examples: "примеры"            # Папка с примерами
  project_pdf: "проект"          # Папка с PDF файлом
  project_pdf_description: "Описание изменений проекта"
```

### 3. Разместите файлы

Создайте структуру папок и разместите файлы:

```
client/
├── замеры/
│   ├── фото_кухня.jpg
│   └── фото_ванная.jpg
├── примеры/
│   ├── пример_кухни.jpg
│   └── пример_светильника.jpg
└── проект/
    └── проект_кухня.pdf
```

### 4. Отправьте проект

```bash
cd client
ruby send.rb
```

Или с указанием другого файла конфигурации:
```bash
ruby send.rb my_project.yaml
```

## Формат конфигурации

### Основные настройки

```yaml
# URL сервера (без слеша в конце)
server_url: "http://localhost:3000"

# API ключ (получите через админку на странице /admin/dashboard)
api_key: "ВАШ_API_КЛЮЧ"

# Данные проекта
project:
  sku: "TEST-001"                # Уникальный артикул (для обновления)
  name: "Название проекта"
  customer: "Заказчик"
  address: "Адрес"
  description: |
    Многострочное описание проекта
    - Пункт 1
    - Пункт 2
```

### Директории с файлами

```yaml
directories:
  # Замеры (изображения) - скрипт автоматически загрузит все файлы из папки
  measurements: "замеры"

  # Примеры (изображения) - скрипт автоматически загрузит все файлы из папки
  examples: "примеры"

  # Проект в формате PDF - скрипт автоматически загрузит первый PDF файл из папки
  project_pdf: "проект"
  project_pdf_description: "Последние изменения проекта"
```

## Получение API ключа

### Через веб-интерфейс

1. Откройте админку: `http://localhost:3000/admin/login`
2. Войдите под учетной записью администратора
3. На дашборде найдите раздел "Ключи для проектов"
4. Введите название ключа
5. Нажмите "Создать"
6. Скопируйте сгенерированный ключ в конфигурацию

### Через консоль (для разработчиков)

```bash
rails runner "ProjectKey.create!(name: 'API ключ', key: SecureRandom.hex(32), active: true)"
```

## Обновление проекта

При повторной отправке с тем же `sku`, проект будет обновлен:
- Текстовые данные заменятся на новые
- Файлы перезагрузятся
- Счетчик обновлений увеличится

**Пример обновления:**
```bash
# Первый запуск - создание проекта
ruby send.rb project_config.yaml

# Второй запуск - обновление проекта (тот же sku)
ruby send.rb project_config.yaml
```

## Важные примечания

### Автоматическая загрузка файлов

- ✅ Скрипт автоматически загружает ВСЕ файлы из указанных папок
- ✅ Замеры и примеры - все файлы из папки
- ✅ Проект (PDF) - первый PDF файл из папки
- ⚠️ Файлы с "Zone.Identifier" в имени игнорируются (Windows)

### Автоматические описания

- 📝 Для файлов описания берутся автоматически из имени файла (без расширения)
- 📄 Для проекта (PDF) можно указать описание отдельно через `project_pdf_description`

**Пример:**
- Файл: `фото_кухня.jpg` → Описание: `фото_кухня`
- Файл: `пример_светильника.png` → Описание: `пример_светильника`

### Уникальный SKU

- 🔑 Используйте уникальный артикул для каждого проекта
- 🔑 SKU используется для обновления существующего проекта
- ⚠️ Если SKU совпадает с существующим - проект будет обновлен, а не создан

### Размеры и типы файлов

- 📏 Учитывайте ограничения на размер файлов при загрузке
- 🖼️ `замеры/` - все файлы будут загружены как замеры (изображения)
- 🎨 `примеры/` - все файлы будут загружены как примеры (изображения)
- 📄 `проект/` - первый PDF файл будет загружен как проект

**Поддерживаемые форматы изображений:**
- JPG / JPEG
- PNG
- GIF
- WEBP
- BMP

## Возможности интерфейса

После отправки проекта пользователь может:

✅ **Карточки изображений** - замеры и примеры отображаются в красивых карточках
✅ **Модальное окно** - клик по карточке открывает увеличенное изображение
✅ **Названия без расширений** - чистые названия файлов без .jpg, .png
✅ **PDF в информации** - PDF проекта отображается в блоке "Информация о проекте"
✅ **Счетчик обновлений** - автоматическое отслеживание количества обновлений проекта
✅ **Поиск по проектам** - быстрый поиск по названию, заказчику, адресу

## Устранение проблем

### Файл конфигурации не найден

**Ошибка:**
```
Ошибка: Файл конфигурации не найден: project_config.yaml
```

**Решение:**
- Убедитесь, что находитесь в директории `client/`
- Или укажите полный путь к конфигурационному файлу
- Проверьте, что файл существует

### API ключ недействителен

**Ошибка:**
```
✗ Ошибка:
{"error": "Invalid API key"}
```

**Решение:**
1. Проверьте, что используете правильный тип ключа (для проектов)
2. Получите новый API ключ в админке
3. Убедитесь, что ключ активен в админке
4. Проверьте, что ключ полностью скопирован

### Файл не найден

**Ошибка:**
```
✗ замеры/фото.jpg - ФАЙЛ НЕ НАЙДЕН!
```

**Решение:**
1. Проверьте, что файл существует в указанной папке
2. Убедитесь, что путь указан относительно директории `client/`
3. Проверьте правильность написания имени файла
4. Убедитесь, что папка существует

### Сервер не доступен

**Ошибка:**
```
✗ Ошибка соединения:
Connection refused
```

**Решение:**
1. Убедитесь, что сервер запущен: `bin/rails server`
2. Проверьте правильность URL в конфигурации
3. Проверьте, что порт 3000 не занят другим приложением
4. Проверьте настройки фаервола

### Неверный тип файла

**Ошибка:**
```
✗ проект/файл.txt - НЕ PDF ФАЙЛ!
```

**Решение:**
- Положите PDF файл в папку `проект/`
- Убедитесь, что файл имеет расширение `.pdf`
- Проверьте, что это действительно PDF файл

## Требования

- **Ruby:** 2.7+
- **Библиотеки:**
  - YAML (входит в стандартную библиотеку)
  - MIME::Types (установите через: `gem install mime-types`)
  - Net::HTTP (входит в стандартную библиотеку)

### Установка MIME::Types

```bash
gem install mime-types
```

Или добавьте в Gemfile:
```ruby
gem 'mime-types'
```

## Продвинутые возможности

### Множественные проекты

Вы можете иметь несколько конфигурационных файлов:

```
client/
├── project_config_kitchen.yaml    # Кухня
├── project_config_bathroom.yaml   # Ванная
├── project_config_living.yaml     # Гостиная
└── ...
```

Отправка:
```bash
ruby send.rb project_config_kitchen.yaml
ruby send.rb project_config_bathroom.yaml
```

### Настройка разных серверов

```yaml
# config_prod.yaml
server_url: "https://production-server.com"
api_key: "PROD_API_KEY"
```

```yaml
# config_dev.yaml
server_url: "http://localhost:3000"
api_key: "DEV_API_KEY"
```

Отправка на разные серверы:
```bash
ruby send.rb config_prod.yaml
ruby send.rb config_dev.yaml
```
